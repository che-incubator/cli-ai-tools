# AGENTS.md — cli-ai-tools

## Overview

Docker image repo for Eclipse Che. Produces `quay.io/che-incubator/cli-ai-tools:latest` —
a UDI-based dev environment bundling AI CLI tools (Aider, RA.Aid), shells (fish/zsh/bash),
and prompt tooling (Starship). **No application source code exists.**

Base image: `quay.io/devfile/universal-developer-image:latest`

## Conventions

### Dockerfile

**Layer discipline:**
- One logical concern per `RUN` block.
- `yum install` → always end with `yum clean all && rm -rf /var/cache/yum` in same RUN.

**User model:**
- `USER root` immediately after `FROM` for all privileged ops.
- `USER 10001` (Che default UID) as **last instruction** before `ENV SHELL`.

**PATH:**
- `ENV PATH="..."` (not `RUN export`). Prepend new dirs; never replace `${PATH}`.

**Python venvs:**
- Isolated venv per tool: `/opt/<name>_venv`.
- Install via `uv pip install --python /opt/<name>_venv/bin/python`.
- Add venv `bin/` to `ENV PATH`.

**Comments:**
- Every major `RUN` block needs a comment explaining *what* and *why*.
- Workarounds must reference upstream issue URLs.

### ra-aid-requirements.txt

- All versions pinned with `==` — no `>=`, `~=`, or unpinned.
- Sorted alphabetically (lowercase).
- Generated by `uv pip freeze` — never hand-edit versions.
- Update workflow: scratch venv → modify → `uv pip freeze > ra-aid-requirements.txt` → sort → commit.

### devfile.yaml

- Schema `2.3.0`, 2-space indent.
- Minimal: image ref + resource limits only. No commands/events/projects.
- Resources as strings with units: `"1Gi"`, `'4'`.

### GitHub Actions

- EPL-2.0 license header block at top of every workflow file.
- Prefer pinning actions to full-length commit SHAs for immutability; major version tags (`@v3`, `@v4`) are acceptable as a minimum. Never use `@latest`.

### Git

- **DCO sign-off required**: `git commit -s -m "message"`.
- `.gitignore`: `.ra-aid/` (session data), `.aider*` (session files).

## Anti-Patterns (DO NOT)

- ❌ Install Python tools globally — use isolated `/opt/<name>_venv`
- ❌ Use `RUN export PATH` — use `ENV PATH` for persistence
- ❌ Skip `yum clean all` after `yum install` — bloats image
- ❌ Reference unverified image tags/package versions — verify via `skopeo list-tags` or `pip index versions` first
- ❌ Guess SHA256 digests — fetch from registry with `skopeo inspect`
- ❌ Hand-edit version numbers in `ra-aid-requirements.txt` — regenerate via freeze
- ❌ Add user-level dotfiles — configure shells system-wide via `/etc/`
- ❌ Commit without `-s` flag — DCO sign-off is mandatory
- ❌ Use `@latest` for GitHub Actions — pin to version tags

## Architecture Notes

- **Shell parity**: Starship configured system-wide for bash (`/etc/profile.d/starship.sh`), zsh (`/etc/zshrc`), fish (`/etc/fish/conf.d/starship.fish`). Config/cache in `/opt/starship/` to avoid home dir issues in containers.
- **Font**: FiraCode Nerd Font v3.4.0 installed system-wide at `/usr/local/share/fonts/FiraCode`.
- **uv**: Installed to `/usr/local/bin` (not cargo default) via `UV_INSTALL_DIR`.
- **chectl**: Installed at `/usr/local/bin/chectl` from `che-incubator.github.io`.
